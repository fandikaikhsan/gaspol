## Cursor Cloud specific instructions

### Project Overview

Gaspol is a Next.js 14 (App Router) adaptive learning platform for Indonesian UTBK exam prep. It uses Supabase (PostgreSQL + Auth + Edge Functions) as its backend, with Tailwind CSS and shadcn/ui for styling.

### Services

| Service | Port | Purpose |
|---------|------|---------|
| Next.js dev server | 3000 | `npm run dev` — main web app |
| Supabase (local) | 54321 (API), 54322 (DB), 54323 (Studio) | `sudo supabase start` from repo root |

### Starting the dev environment

1. **Start Docker** (if not running): `sudo dockerd &`
2. **Start local Supabase**: `cd /workspace && sudo supabase start` — applies all migrations automatically. The seed file (`supabase/seed.sql`) has invalid UUIDs (e.g. `q1111111-...`) and will fail; seed is disabled in `config.toml`.
3. **Start Next.js**: `npm run dev`

The `.env.local` file should point to the local Supabase instance:
- `NEXT_PUBLIC_SUPABASE_URL=http://127.0.0.1:54321`
- Use keys from `sudo supabase status -o json` (`ANON_KEY`, `SERVICE_ROLE_KEY`)

### Key commands

- **Lint**: `npm run lint` (warnings only, no errors)
- **Test**: `npm test` (Vitest, 77 tests across 2 files)
- **Dev**: `npm run dev`
- **Build**: `npm run build` (TypeScript and ESLint errors are ignored via `next.config.js`)

### Gotchas

- `supabase/config.toml` does not exist in the repo; it is generated by `supabase init --force`. The seed must be disabled (`enabled = false` under `[db.seed]`) because `seed.sql` uses non-hex UUID prefixes.
- Docker runs inside a Firecracker VM; requires `fuse-overlayfs` storage driver and `iptables-legacy`. See the Docker setup in the environment snapshot.
- Supabase commands require `sudo` because Docker socket ownership.
- AI features (Anthropic/OpenAI/Gemini) are optional; the core student flow works without API keys.
- After `supabase start`, local auth has email confirmation disabled, so signup works immediately.
